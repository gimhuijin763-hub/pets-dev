# PRD&#x20;

## 반려동물 입양 신청 관리 MVP

---

## 1. 프로젝트 개요

- **프로젝트명**: ptes
- **목표**: 입양 가능한 반려동물을 조회하고, 사용자가 입양 신청서를 제출하면, 관리자가 신청 상태를 관리할 수 있는 최소 기능의 웹 서비스를 만든다.
- **배경**: 보호소/입양센터는 입양 정보를 여러 도구로 나눠 관리하는 경우가 많아 비효율이 크다. 해커톤 MVP에서는 가장 핵심적인 흐름인 **동물 조회 → 신청서 제출 → 관리자 상태 관리**를 빠르게 연결한다.

---

## 2. 타겟 사용자

- **타겟**

  - 반려동물 입양 희망자
  - 보호소/입양센터 관리자

- **사용자 니즈**

  - 입양 희망자는 입양 가능한 동물을 쉽게 찾고 바로 신청하고 싶다.
  - 관리자는 신청 내역을 한곳에서 보고 상태를 빠르게 변경하고 싶다.

---

## 3. 핵심 기능

### 필수 기능

- 입양 가능 동물 목록
- 동물 상세 페이지
- 입양 신청 폼 제출
- 관리자 신청 목록 확인
- 신청 상태 변경

### 제외 기능

- 로그인/회원가입
- 결제
- 채팅
- 의료 기록 상세 관리
- 임시보호 관리
- 알림 자동화
- 다기관 운영

---

## 4. 기능 상세 설명

| 기능명       | 설명                     | 입력/출력 데이터              | 우선순위 |
| --------- | ---------------------- | ---------------------- | ---- |
| 동물 목록     | 입양 가능한 동물 리스트 표시       | 동물 데이터 / 목록 출력         | 높음   |
| 동물 상세     | 이름, 나이, 성별, 설명, 사진 표시  | 동물 ID / 상세 정보 출력       | 높음   |
| 입양 신청 폼   | 신청자 정보 입력 및 제출         | 이름, 연락처, 신청 사유 / DB 저장 | 높음   |
| 관리자 신청 목록 | 전체 신청 내역 조회            | 신청 데이터 / 목록 출력         | 높음   |
| 신청 상태 변경  | 접수, 검토 중, 승인, 거절 상태 관리 | 상태값 입력 / 상태 출력         | 높음   |

---

## 5. 성능 요구사항

- 메인 페이지 로딩 2초 이내
- 모바일/PC 웹 지원
- 신청서 제출이 정상 저장될 것
- 관리자 페이지에서 상태 변경이 즉시 반영될 것

---

## 6. 제약사항

- **개발 기간**: 1일
- **기술 스택**: Next.js, TypeScript, Tailwind CSS, Supabase, Vercel
- **예산**: 0원
- **개발 원칙**
  - 기능 수 최소화
  - 화면 수 최소화
  - 데모 가능한 흐름 우선
  - 디자인보다 동작 우선

---

## 7. 성공 지표

- 동물 목록이 정상 조회된다
- 상세 페이지가 열린다
- 신청서 제출이 된다
- 관리자 페이지에서 신청 내역이 보인다
- 상태 변경이 정상 작동한다
- 발표 데모가 끊기지 않고 끝까지 진행된다
