# Active Context

## 현재 작업 범위

### 완료된 주요 작업
1. **라우팅 구조 정리**
   - `/` → 랜딩 페이지 (서비스 소개)
   - `/animals` → 동물 목록 페이지
   - `/animals/[id]` → 동물 상세 페이지
   - `/animals/[id]/apply` → 입양 신청 폼
   - `/admin` → 관리자 신청 관리 페이지

2. **핵심 기능 구현**
   - 동물 목록 조회 및 타입별 필터링 (전체/개/고양이)
   - 동물 상세 정보 표시 (이미지, 이름, 나이, 성별, 설명)
   - 입양 신청 폼 (유효성 검증 포함)
   - 관리자 신청 목록 및 상태 변경
   - 실시간 통계 대시보드

3. **UI/UX 개선**
   - 랜딩 페이지 디자인 및 콘텐츠 작성
   - 반응형 레이아웃 (모바일/태블릿/데스크톱)
   - 상태 뱃지 컴포넌트
   - 이미지 로드 실패 시 fallback 처리

4. **데이터 관리**
   - Supabase Postgres 기반 데이터 저장
   - API Route Handler를 통한 서버 사이드 검증
   - 신청서 저장 시 서버 유효성 검증

5. **환경 설정**
   - Supabase 환경변수 및 서버 전용 키 설정
   - 관리자 인증용 비밀번호/쿠키 시크릿 추가
   - TypeScript 타입 정의
   - Tailwind CSS 커스텀 스타일

## 현재 제약사항

### 기술적 제약
- Supabase 서비스 롤 키가 누락되면 서버 API가 동작하지 않음
- 이미지는 외부 URL (Unsplash) 의존
- 관리자 인증은 단일 비밀번호 기반 (권한 분리 없음)

### 기능적 제약
- 일반 사용자 로그인 없음 (입양 신청용)
- 관리자 인증은 단일 비밀번호 기반
- 동물 데이터 수정 불가 (관리자 CRUD 미구현)
- 실시간 알림 없음
- 파일 업로드 불가
