# Project Brief

## 프로젝트명
**ptes** (Pet Adoption Management System)

## 프로젝트 목적
보호소와 입양센터가 입양 가능한 반려동물 조회부터 입양 신청 접수, 관리자 상태 관리까지 하나의 흐름으로 연결된 웹 서비스를 제공하여 분산된 입양 프로세스의 비효율을 해소한다.

## 핵심 가치 제안
- **통합된 흐름**: 동물 조회 → 신청서 제출 → 관리자 상태 관리를 하나의 플랫폼에서 처리
- **빠른 운영**: 여러 도구를 오가지 않고 한 곳에서 신청 내역 확인 및 상태 변경
- **단순한 UX**: 로그인 없이 핵심 기능에 집중한 최소 기능 MVP

## 프로젝트 범위

### 포함 기능
1. **입양 가능 동물 목록 조회**
   - 동물 타입별 필터링 (전체/개/고양이)
   - 카드 형태 목록 UI
   - 입양 상태 뱃지 표시

2. **동물 상세 페이지**
   - 이름, 나이, 성별, 설명, 사진 표시
   - 입양 신청 버튼

3. **입양 신청 폼**
   - 신청자 정보 입력 (이름, 연락처, 이메일, 신청 사유)
   - 유효성 검증
   - 제출 완료 피드백

4. **관리자 신청 관리**
   - 전체 신청 내역 조회
   - 상태별 필터링 (전체/접수/검토 중/승인/거절)
   - 신청 상태 변경 (드롭다운)
   - 실시간 통계 카드

5. **랜딩 페이지**
   - 서비스 소개 및 가치 제안
   - 문제 정의 및 해결 방식
   - 핵심 기능 설명
   - FAQ

### 제외 기능
- 로그인/회원가입
- 결제
- 채팅
- 의료 기록 상세 관리
- 임시보호 관리
- 알림 자동화
- 다기관 운영

## 핵심 요구사항

### 기능 요구사항
- 동물 목록이 정상 조회되어야 함
- 상세 페이지가 정상 열려야 함
- 신청서 제출이 정상 저장되어야 함
- 관리자 페이지에서 신청 내역이 보여야 함
- 상태 변경이 즉시 반영되어야 함

### 성능 요구사항
- 메인 페이지 로딩 2초 이내
- 모바일/PC 웹 반응형 지원
- 신청서 제출 즉시 저장
- 관리자 상태 변경 즉시 반영

### 비기능 요구사항
- 발표 데모가 끊기지 않고 끝까지 진행되어야 함
- 디자인보다 동작 우선
- 화면 수 최소화
- 기능 수 최소화

## 제약사항
- **개발 기간**: 1일
- **기술 스택**: Next.js 14, TypeScript, Tailwind CSS, Supabase (선택), Vercel
- **예산**: 0원
- **개발 원칙**: 
  - 기능 수 최소화
  - 화면 수 최소화
  - 데모 가능한 흐름 우선
  - 디자인보다 작동 우선

## 성공 지표
1. 동물 목록이 정상 조회된다
2. 상세 페이지가 열린다
3. 신청서 제출이 된다
4. 관리자 페이지에서 신청 내역이 보인다
5. 상태 변경이 정상 작동한다
6. 발표 데모가 끊기지 않고 끝까지 진행된다

## 타겟 사용자
- **1차 사용자**: 보호소/입양센터 관리자
- **2차 사용자**: 반려동물 입양 희망자

## 한 줄 요약
보호소의 분산된 입양 신청 과정을, 동물 조회부터 신청 접수와 상태 관리까지 하나의 흐름으로 연결한 MVP입니다.
